Task 7

Thin-lens virtual imaging (object inside focal length) with ray tracing and inverse mapping

Overview

Place the object between the lens and its focal point (u < f). The lens forms a virtual, upright, magnified image on the object’s side. This app renders that virtual image by mapping pixels and shows ray tracing so you can see how the back‑projected principal rays meet at the virtual image location.

Key Theory

Thin‑lens relation: 1/u + 1/v = 1/f. With the lens at x = 0 and the object at u > 0 on the right, if u < f there is no real intersection of refracted rays on the left. Instead, extending the refracted rays back to the right gives a virtual image at v = + fu/(f − u) > 0. The transverse magnification is m = v/u > 1 (upright, enlarged).

Ray tracing connects directly to this result. Two principal rays from the object top are drawn: (1) the central ray through the lens centre, which is undeviated; (2) the ray parallel to the axis that refracts as if it came from the focal point on the far side. These emerging rays diverge, but their straight‑line extensions back into the object half‑space intersect at the virtual image point (x = v, y = m·y₀). The code both renders the image via inverse mapping and marks that theoretical intersection to show consistency.
