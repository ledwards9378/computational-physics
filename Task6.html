Task 6

Thin-lens imaging: interactive real-image rendering with inverse mapping and a focal-length slider

Overview

Load an object image on the right of a converging lens at x = 0, adjust the focal length, and move the object. The program computes where a real image forms on the left (when u > f) and renders it using a physically consistent inverse mapping. It also annotates the theoretical image distance and magnification.

Key Theory

The thin-lens equation relates object distance u and image distance v (measured from the lens): 1/u + 1/v = 1/f. With the sign convention used here, the lens is at x = 0, the object sits at x = u > 0, and a real image appears to the left at v < 0 when u > f. The transverse magnification is m = −v/u: real images are inverted (m < 0) and scaled by |m|.

For rendering, we use an inverse mapping from screen coordinates (X, Y) on the image side back to source coordinates (x, y) on the object, derived from similar triangles and the magnification relation. This guarantees a single sample per output pixel and avoids gaps compared with naive forward mapping. When u ≤ f the rays do not converge to a real image; the program suppresses the left-side render and indicates that a virtual image would lie on the right.
